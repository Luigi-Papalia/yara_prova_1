Vulnerable JavaScript Microservice

This page explains how to use the intentionally vulnerable endpoints of this microservice and demonstrates potential attacks.

/greet

This endpoint accepts a 'name' parameter via a query string and displays it in the HTML output.

Vulnerability: Cross-Site Scripting (XSS)

Usage

Access the endpoint with a name parameter:

/greet?name=YourName

Attack Example

Inject JavaScript code to execute in the user's browser:

/greet?name=<script>alert('XSS Attack!')</script>

/calculate

This endpoint evaluates a mathematical expression provided in the 'expr' parameter via a query string.

Vulnerability: Code Injection

Usage

Access the endpoint with an expression parameter:

/calculate?expr=2+2

Attack Example

Execute arbitrary JavaScript code on the server:

/calculate?expr=process.exit(1)

Disclaimer: This microservice is intentionally vulnerable for demonstration purposes. Do not deploy this code in a production environment.
